Search Engine Optimization (SEO):

SEO stands for Search Engine Optimization. It is to improve the visibility of your website on search engines.

Listed below are 5 ways to improve the visibility of your website on search engines:

  • Improve page load time
  • Mobile friendly, responsive design
  • Keyword phrases (in the Meta tag). Ask other people what words/phrases they would use. Check Google trends
  • Name your pages (use title tag)
  • Use “description” tag
Web Publishing:

Web hosting is where the website documents actually store.

Hosting should address:

In order for you website to be available on the Internet it needs to be placed on a computer attached to the internet. While it is possible to host your website yourself, it requires dedicating a powerful PC server to that purpose and making sure it is running and accessible to the internet 24 hours a day. If you want to have interactive functionality, (such as database access or, forms processing) or the ability to run server side programs (usually written in ASP/ASP.NET or PHP) you will need to have Web Server software installed.

Costs will be based on the amount of storage and bandwidth (amount of traffic in and out of your site) you require, and add-on services such as e-mail accounts.---------Promotion strategy & marketing should address

Marketing is a mix of price, product, place and promotion. So you can tell someone you have a great web site (Marketing) and the site can be found by searching for the company name in Google/Bing (promotion).

Web Performance & Maintenance:

hen maintaining a website,

there are 5 ways to reduce page loading time:

  • Optimize images to no longer than screen resolution and save it as a compressed file;
  • Remove unnecessary widgets;
  • Use HTTP compression: compress data before sending out from web server to client devices.
  • Placing CSS at the top AND script reference at the bottom or in external files;
  • Optimize DNS lookups (website URL)

    • Web security:

    Listed below are 5 key threats that need to be considered by web developers:

  • User input: Don't trust user input because they can add malicious JavaScript into a comment box;
  • Phishing: A fake version of your web site;
  • XSS: Cross-site scripting - common attack vector that injects malicious code into a vulnerable web application;
  • CDRF: Cross domain request forgery (Posting a form from one website to another) - an attack that forces an end user to execute unwanted actions on a web application in which they're currently authenticated;
  • Click-jacking: tricking a user into clicking on something different from what the user perceives they are clicking on;
